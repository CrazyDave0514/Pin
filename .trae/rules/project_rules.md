> **每次执行任务前，必须先阅读本文件和 TASK_STATUS.md。**

- **前端**：uni-app（Vue3 + Composition API + TypeScript）

- **部署**：GitHub Pages（前端）、阿里云（后端）

## 多AI协作方式

本项目由多个独立对话的 AI 协作完成，通过 **共享文件** 协调：

1. **每次任务开始前**，必须先阅读 `PROJECT_RULES.md` 和 `TASK_STATUS.md`
2. **领取任务**：从 `TASK_STATUS.md` 中认领状态为 `待开始` 的任务，改为 `进行中`
3. **完成任务**：更新 `TASK_STATUS.md` 中任务状态为 `已完成`，填写交付物路径
4. **产出物**：代码提交到 GitHub 对应分支，文档存放到飞书对应目录
5. **阻塞沟通**：遇到阻塞问题，在 `TASK_STATUS.md` 的备注中说明，项目Owner会协调

**状态值**：`待开始` → `进行中` → `已完成` / `阻塞`

## 工作流程

```
需求确认（Owner）→ 技术方案（AI后端）→ 设计确认（Owner）
→ 并行开发（AI前端 + AI后端）→ 联调 → 测试（AI测试）→ 验收（Owner）→ 发布
```

**原则**：上一阶段确认后才能进入下一阶段。验收不通过打回开发阶段。

## 代码规范

- **分支**：`main`（生产）、`develop`（开发）、`feature/版本-功能`
- **Commit**：`feat:` / `fix:` / `docs:` / `refactor:` / `test:` / `chore:`
- **Review**：提 PR → 交叉 Review → 合并

## 文件规范

- **命名**：`Pin - [版本号] [文档类型]`（如 `Pin - V0.2.0 需求文档`）
- **存放**：所有文档存飞书，严禁仅存本地
- **飞书目录**：01-需求文档 / 02-设计文档 / 03-技术文档 / 04-测试文档
- **临时文件**：所有临时文件、草稿、脚本必须放在 `/sessions/.../work/` 目录，**严禁放在 workspace 根目录**
- **禁止行为**：不得在 workspace 根目录创建 `.md`、`.js`、`.txt` 等临时文件
