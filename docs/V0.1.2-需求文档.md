## 二、P0 需求（重点）

### 2.1 画布保存数据修复

**优先级**：P0（严重 BUG）

**问题描述**：
保存的空白画布作品，再次点开是空白内容，数据未正确保存。

**根因分析**：
1. 保存时 `canvasData` 中的 `beads` 数组可能未正确序列化
2. 打开项目时未正确从 `pin_projects` 中读取 `canvasData` 并还原到画布
3. `canvasData` 对象引用问题：保存时存储的是引用而非深拷贝

**修复方案**：
1. 保存时确保 `canvasData` 深拷贝（`JSON.parse(JSON.stringify(...))`）
2. 打开项目时正确读取 `canvasData.beads` 数组并还原
3. 验证保存→退出→重新打开的完整流程

**验收标准**：
- [ ] 创建空白画布 → 放置拼豆 → 保存 → 退出 → 重新打开，拼豆数据完整还原
- [ ] 保存后项目列表显示正确的缩略图信息
- [ ] 多次保存不会产生重复项目

### 2.2 画布编辑器移动端 PWA 适配

**优先级**：P0

**问题描述**：
画布编辑页面的【取消】和【保存】按钮点不着，底部工具栏可能被遮挡。

**修复方案**：

#### 2.2.1 导航栏安全区域适配
- 导航栏顶部增加 `padding-top: var(--status-bar-height)` 适配状态栏
- 导航栏高度使用 `calc(88rpx + var(--status-bar-height))`
- 确保取消/保存按钮在所有设备上可点击
- 按钮点击热区不小于 88rpx × 88rpx

#### 2.2.2 底部工具栏安全区域适配
- 底部颜色面板已有 `env(safe-area-inset-bottom)`，确认工具栏同样适配
- 工具栏按钮点击区域不小于 88rpx × 88rpx
- 保存/取消按钮增加点击热区（padding 扩大）

**验收标准**：
- [ ] 取消按钮在 iPhone SE / iPhone 15 Pro Max 均可正常点击
- [ ] 保存按钮在所有测试设备上可正常点击
- [ ] 底部工具栏不被系统手势条遮挡
- [ ] 导航栏不被刘海屏/状态栏遮挡

### 2.3 画布触摸体验优化

**优先级**：P0

**问题描述**：
1. 画布上下滑动时整体页面会一起滑动，体验差
2. 点击画笔放置像素有延迟感

**修复方案**：

#### 2.3.1 阻止画布区域页面滚动
- 画布容器使用 `@touchmove.prevent` 阻止默认滚动行为
- 或使用 `catchtouchmove`（小程序）阻止事件冒泡
- 画布区域外（工具栏、颜色面板）保持正常滚动

#### 2.3.2 触摸响应优化
- 移除 `handleTouch` 中多余的 `e.target.getBoundingClientRect()` 调用
- 使用 `uni.createSelectorQuery()` 替代 `document.querySelector()`（跨平台兼容）
- 触摸移动时使用 `requestAnimationFrame` 节流，减少重绘频率
- 填充工具（fill）仅在 `touchstart` 时触发，不在 `touchmove` 时重复触发

#### 2.3.3 画布缩放与拖拽
- 支持双指缩放画布（pinch zoom）
- 支持单指拖拽画布（拖动工具模式下）
- 缩放范围：25% ~ 400%

**验收标准**：
- [ ] 在画布上绘制时页面不会跟着上下滑动
- [ ] 放置像素响应流畅，无明显延迟
- [ ] 双指可缩放画布
- [ ] 拖动工具模式下可平移画布
- [ ] 工具栏和颜色面板区域可正常滚动

## 三、P1 需求

### 3.1 首页假数据填充（300+ 条）

**优先级**：P1

**需求描述**：
首页需要至少 300 条假数据做展示，营造社区内容丰富的氛围。

**数据字段要求**：

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 唯一标识 | artwork_001 |
| name | 作品名称 | 可爱的小猫咪拼豆 |
| coverImage | 封面图 | 像素画风格图片 |
| creatorName | 创作者名称 | 拼豆达人小王 |
| likes | 点赞数 | 128 |
| points | 积分数 | 5 |
| createdAt | 创建时间 | 随机近 30 天内 |
| tags | 标签 | ['动物', '可爱'] |
| viewCount | 浏览量 | 256 |
| useCount | 使用量 | 32 |

**封面图方案**（参考图3）：
- 像素画风格封面
- 使用占位图服务生成像素画风格图片
- 或从开源像素画库获取示例图片
- 封面尺寸：400×400 像素

**数据生成规则**：
- 创作者名称：中文昵称，20~30 个不同用户名循环使用
- 作品名称：从拼豆主题词库组合生成（动物/植物/动漫/食物/风景等）
- 点赞数：0~5000 随机，热门作品偏高
- 积分数：1~50 随机
- 创建时间：近 30 天内随机分布
- 标签：从预设标签库中随机选取 1~3 个

**数据存储**：
- 数据写入 `uni.getStorageSync('pin_artworks')`
- 同时导出一份到飞书文档备份

**验收标准**：
- [ ] 首页展示至少 300 条作品数据
- [ ] 数据包含用户名、点赞数、积分数、封面、创建时间
- [ ] 封面图片可正常加载显示（像素画风格）
- [ ] Tab 切换（推荐/热门/最新）有实际过滤逻辑
- [ ] 数据已备份到飞书文档

## 四、P2 需求

### 4.1 编辑资料功能

**优先级**：P2

**需求描述**：
支持修改用户个人信息，包括昵称、头像、出生年月日、性别。

**入口方式**：
1. **入口一**：在"我的"页面，点击用户头像/昵称区域进入编辑资料页
2. **入口二**：在"我的"页面，点击"更多设置" → "编辑资料"进入编辑资料页
- 两个入口功能完全一致

**编辑资料页字段**：
- 头像：点击可更换（复用现有裁切功能）
- 昵称：文本输入，2~20 字符
- 性别：单选（男/女/保密），未设置时默认"保密"
- 出生日期：日期选择器

**交互说明**：
1. 点击保存，数据写入 `pin_user` 存储
2. 返回"我的"页面，昵称和头像实时更新

**数据结构**：
```json
{
  "uid": "user_xxx",
  "username": "拼豆达人",
  "avatar": "/static/avatar_xxx.png",
  "gender": "male",
  "birthday": "2000-01-01",
  "createdAt": 1234567890
}
```

**验收标准**：
- [ ] 可通过头像/昵称入口进入编辑资料
- [ ] 可通过更多设置-编辑资料入口进入
- [ ] 可修改昵称并保存
- [ ] 可更换头像并保存
- [ ] 可选择性别（未设置时默认保密）
- [ ] 可选择出生日期
- [ ] 保存后"我的"页面昵称和头像实时更新

### 4.2 默认头像与性别标识

**优先级**：P2

**需求描述**：
"我的"页面未登录和登录后都需要有默认头像，并增加性别标识。

- 显示默认头像（中性风格）

- 使用 SVG 或 PNG 绘制简洁的拼豆风格默认头像
- 男性默认：蓝色调
- 女性默认：粉色调
- 通用默认：灰色调

**验收标准**：
- [ ] 未登录时显示默认头像
- [ ] 已登录无自定义头像时显示默认头像
- [ ] 昵称旁显示性别标识图标
- [ ] 性别标识颜色区分（男蓝/女粉/保密灰）
