PIN 本地存储服务层收口说明
1. 现状 key 映射
保留全部既有 pin_* key，不变更存储名称，只收口访问入口。
登录/用户：pin_user、pin_user_list
积分：pin_points、pin_points_records
项目：pin_projects、pin_folders、pin_recent_imports
社区作品：pin_artworks、pin_artworks_version
互动关系：pin_liked_artworks、pin_favorited_artworks、pin_purchased_artworks、pin_followed_creators
搜索/设置：pin_search_history、pin_settings
注销联动清理：pin_favorite_colors
2. 服务层结构
src/services/pin/storage-adapter.ts：统一底层读写，提供 UniStorageAdapter 与 MemoryStorageAdapter。
src/services/pin/provider.ts：定义 PinDataProvider 领域契约。
src/services/pin/local-provider.ts：默认本地 provider，继续落本地存储。
src/services/pin/aliyun-provider.ts：预留阿里云 API provider 骨架。
src/services/pin/index.ts：统一暴露 Promise 风格服务接口。
3. 暴露服务
authService：当前用户、最近登录用户、注册、快速登录、资料更新、登录态判断、设置项读取写入、清缓存、退出登录、注销账号。
projectService：项目列表、文件夹列表、单项目、最近导入、搜索历史、项目搜索数据源聚合、项目导出作者名读取。
communityService：社区作品读取、作品更新、点赞收藏关注切换、我的点赞收藏、发布同步、下架同步、互动状态聚合。
purchaseService：购买可用性判断、积分扣减、购买记录写入、已购买列表写入、社区销量自增。
pointsService：积分余额读取写入、默认积分初始化、积分记录读取、默认展示记录初始化、明细追加。
4. 页面迁移范围
登录/用户链路：src/pages/login/index.vue、src/pages/mine/index.vue、src/pages/mine/avatar-crop.vue、src/pages/profile/edit.vue、src/pages/settings/index.vue
积分/社区链路：src/pages/points/index.vue、src/pages/index/index.vue、src/pages/artwork-detail/index.vue、src/pages/mine/collection.vue
项目链路：src/pages/search/index.vue、src/pages/blueprint-import/index.vue、src/pages/project/index.vue、src/pages/canvas-editor/index.vue
说明：画布编辑器保留一段与蓝图导入临时 key 相关的 uni.getStorageSync(options.key) 逻辑，它不是 pin_* 存储，不属于本次服务层收口范围。
说明：src/utils/community.ts 已改为通过统一 storage adapter 访问数据，不再直接调用 uni.*Storage*。
5. 阿里云切换位
后续只需在 AliyunPinDataProvider 中实现与 PinDataProvider 对齐的方法，并在 createPinServices 中切换 provider 来源，页面继续调用既有 Promise 服务接口即可。
页面层当前不感知 uni、本地 key 名、本地与远端数据源差异。
